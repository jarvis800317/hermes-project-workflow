# 繼續工作技能

接續先前工作，快速恢復現場。

## SOP

### 步驟 1：確認專案

從對話脈絡推斷專案名稱，預設路徑：
- 專案總資料夾：`~/我的雲端硬碟/Jarvis專案/`
- Obsidian 工作日誌：`~/我的雲端硬碟/2ndBrain/工作日誌/`

若無法確認，詢問：「請告訴我專案名稱。」

### 步驟 2：讀取工作日誌

依序檢查：
1. `~/我的雲端硬碟/2ndBrain/工作日誌/{專案名}.md`
2. `~/我的雲端硬碟/Jarvis專案/{專案名}/工作日誌.md`

若都找不到，回應：「找不到『{專案名}』的工作日誌。可能是新專案，請問要繼續哪個專案？」

### 步驟 3：摘要進度

從工作日誌中提取：
- `上次做到哪`：最後動作、已完成檔案
- `最近更動紀錄`：最近 3-5 筆記錄
- `踩坑筆記`：待注意的事項

用口語化方式回應，範例：

```
📋 專案：桃林鐵路工具
上次做到：完成座標獵人遊戲，已發布至 GitHub Pages
目前進度：100%
待確認：鋼筋遮蔽演算法需優化

🕳️ 待注意：座標系統用 (-5,5)，與地圖資料不同，需做轉換

📍 建議下一步：著手處理座標轉換模組
```

### 步驟 4：詢問意圖

「請問要繼續做什麼？」

## 特殊情況

| 情況 | 回應 |
|------|------|
| 無工作日誌 | 「這是乾淨的開始，請告訴我你要做什麼工具或任務。」 |
| 專案已完成的 | 「這個專案已完成。上次做到：{結案狀態}。要開新專案嗎？」 |
| 多日未繼續 | 「距離上次已 {N} 天。工作日誌顯示：{摘要}。要繼續嗎？」 |

## 不該做的事

- ❌ 未讀工作日誌就假設進度
- ❌ 自行發起需要长时间运行的工作（如下載大型檔案），應先確認意圖
