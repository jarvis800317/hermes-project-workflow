# 初始化新專案

快速建立一個新專案的基礎結構。

## 觸發時機

使用者說：
- 「開新專案」
- 「新建專案」
- 「我想做一個 XXX 工具」

## SOP

### 步驟 1：詢問專案名稱

若使用者只說「開新專案」，詢問：「專案叫什麼名字？」

專案命名規則：
- 英文小寫 + 連字號（例：`coordinate-hunter`、`taolin-tool`）
- 不需要 .md 或 .html 副檔名

### 步驟 2：建立專案結構

```bash
# 建立專案資料夾
mkdir -p ~/我的雲端硬碟/Jarvis專案/projects/{專案名}

# 在 Obsidian 建立工作日誌
mkdir -p ~/我的雲端硬碟/2ndBrain/工作日誌
```

### 步驟 3：初始化 Git + GitHub

```bash
cd ~/我的雲端硬碟/Jarvis專案/projects/{專案名}
git init
gh repo create {專案名} --public --source=. --push
```

### 步驟 4：建立 CLAUDE.md

在專案資料夾建立 `CLAUDE.md`：

```markdown
# {專案名}

## 專案概述
（描述這個工具做什麼）

## 工作狀態
- 進度：0%（剛初始化）
- 開始日期：{今天}
- GitHub：https://github.com/jarvis800317/{專案名}

## 工作模式
- 結束工作：說「收工」同步進度
- 接續工作：說「繼續」讀取工作日誌

## 技術棧
（待填寫）

## 待辦事項
（待填寫）
```

### 步驟 5：建立工作日誌

在 `~/我的雲端硬碟/2ndBrain/工作日誌/{專案名}.md` 建立：

```markdown
# {專案名} 工作日誌

> 📌 進度日誌。每次收工時更新。

## 上次做到哪

**最後動作**：初始化專案
**完成檔案**：CLAUDE.md
**對話脈絡**：剛建立專案，還沒有實質進度

## 最近更動紀錄

| 日期 | 變更摘要 | GDrive | Obsidian | GitHub |
|------|----------|--------|----------|--------|
| {今天} | 初始化專案 | ✅ | ✅ | ✅ |

## 踩坑筆記

（待記錄）
```

### 步驟 6：建立 .gitignore

```bash
# 敏感資料
.env
*.key
credentials.*

# 系統檔
.DS_Store
*.tmp

# Node.js（若需要）
node_modules/
package-lock.json
```

### 步驟 7：報告完成

```
✅ 專案「{專案名}」已初始化完成！

📁 位置：~/我的雲端硬碟/Jarvis專案/projects/{專案名}/
🐙 GitHub：https://github.com/jarvis800317/{專案名}
📝 工作日誌：~/我的雲端硬碟/2ndBrain/工作日誌/{專案名}.md

下一步：請告訴我你想做什麼功能？
```

## 後續工作節奏

| 你說的話 | 我會做的事 |
|---------|-----------|
| 「我要做 XXX 功能」 | 實作功能 |
| 「收工」 | 三方同步 |
| 「繼續」 | 讀取進度繼續工作 |
